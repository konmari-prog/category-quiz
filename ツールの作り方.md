# 自分専用の診断ツールの作り方（初心者向けガイド）

プログラミングの知識がゼロでも、AIの力を借りれば自分だけのオリジナル診断ツールは数分で作れます。このガイドでは、その最短ルートを解説します。

---

## 1. AIへの伝え方

まず、Claude 3.5 SonnetやChatGPTなどのAIに、以下をコピーして伝えてみてください。

```text
ココナラの診断ツールを作りたいです。HTML1ファイルで完結させてください。
要件は、8つの質問に答えると、自分にぴったりの出品カテゴリを3つ提案してくれるものです。
デザインは清潔感のあるプロフェッショナルな雰囲気にして、レスポンシブ対応してください。
```

---

## 2. AIが作りやすくなる「要件定義」のコツ

AIに以下の情報を追加で伝えると、より理想に近いツールが完成します。

- **質問の内容**: 何を基準にカテゴリを選びたいか
- **判定タイプ**: いくつのタイプ（例：8タイプ）に分類したいか
- **デザインの好み**: 好きな色や、参考にしたいサイトの雰囲気
- **必要な機能**: 「最後にXでのシェアボタンが欲しい」など

---

## 3. HTMLを公開する方法

AIが生成したコードは、そのままでは自分のコンピューターの中でしか見られません。ネット上で誰でもアクセスできるようにするために「ホスティング」という作業が必要です。

今回は、無料で使えて最も信頼できる **「GitHub Pages」** を使った方法を紹介します。

---

## 4. GitHubページで無料公開する手順

1. **GitHubアカウントを作成**: [GitHub公式サイト](https://github.com/)で登録します。
2. **リポジトリ（箱）を作る**: 「New Repository」ボタンを押し、プロジェクト名を付けて作成します。
3. **ファイルをアップロード**: 作成した箱の中に、AIが書いたHTMLファイルをアップロードします。
4. **公開設定をオンに**: 「Settings」タブ > 「Pages」メニューを開き、公開ブランチを選択して保存します。

数分待つと、自分だけのURL（例：https://あなたの名前.github.io/リポジトリ名/）が発行されます。

---

## 5. GitHubページで「できること・できないこと」

初心者の方が迷いやすいポイントをまとめました。

| 項目 | GitHub Pagesでできること | GitHub Pagesでできないこと |
| :--- | :--- | :--- |
| **コンテンツ** | 診断ツール、ポートフォリオ、紹介サイト | オンラインショップ、会員制サイト |
| **データ保存** | 保存機能なし（画面表示のみ） | 診断者データのデータベース保存 |
| **コスト** | **完全無料**（容量内） | サーバー費用などは一切不要 |
| **セキュリティ** | HTTPS（鍵マーク）が標準で付く | サーバー側の個人情報処理 |

---

## まとめ

まずはAIに「作って」と話しかけるところから始めてみましょう。コードを1行も書かなくても、素晴らしいツールを世界中に公開することができます！
