# 自分専用の診断ツールの作り方（初心者向けガイド）

---

## 📌 1. AIへの伝え方

プログラミングの知識がなくても大丈夫です。ClaudeやChatGPTなどのAIに、まずは以下の指示（プロンプト）をそのまま伝えてみましょう。

> **AIへの指示例:**
> 「ココナラの診断ツールを作りたいです。HTML1ファイルで完結させてください。
> 要件は、8つの質問に答えると、自分にぴったりの出品カテゴリを3つ提案してくれるものです。
> デザインは清潔感のあるプロフェッショナルな雰囲気にして、レスポンシブ対応してください。」

---

## 💡 2. AIが作りやすくなる要件定義のコツ

AIに以下の情報を追加で伝えると、あなたが思い描くツールに一歩近づきます。

- **判定タイプ**: いくつの「タイプ（結果）」を用意したいか
- **デザインの好み**: 好きな色や、参考にしたいサイト
- **必須機能**: 「診断後にX（Twitter）でシェアさせたい」など

---

## 🌐 3. HTMLを公開する方法

AIが書いたコードを保存しただけでは、まだあなたのパソコンの中でしか見られません。
インターネットを通じて世界中に公開するために、**ホスティング（置き場所）**を用意します。

今回は、無料で使えて最も有名な **「GitHub Pages」** を使った方法を推奨します。

---

## 🚀 4. GitHubページで無料公開する手順

### ① GitHubアカウントを作成
- [GitHub公式サイト](https://github.com/)で登録（無料）

### ② リポジトリ（公開用の箱）を作成
- 画面右上の「**+**」 ＞ 「**New repository**」
- **Repository name**: 好きな名前を英数字で（例：`coconala-tool`）
- **重要**: 「**Public**」を選択（無料で公開するための必須条件です）
- 「Create repository」をクリック

### ③ ファイルをアップロード
- 「uploading an existing file」から、AIが書いた `index.html` をアップロード。
- 「Commit changes」で保存。

### ④ 公開設定をオンにする
- 上部「**Settings**」 ＞ 左サイドバー「**Pages**」をクリック。
- **Branch** を「**main**」、フォルダを「**/(root)**」に設定して「**Save**」。

✅ **数分待つとURLが発行され、自分のサイトが公開されます！**

---

## 📋 5. GitHub Pagesで「できること・できないこと」

| 項目 | GitHub Pagesでできること | GitHub Pagesでできないこと |
| :--- | :--- | :--- |
| **得意** | 診断ツール、紹介サイト、LP、ポートフォリオ | 会員登録、ログイン、決済機能 |
| **データ** | 画面の表示、画像のダウンロード | 診断データの保存（DB）、個人情報の収集 |
| **費用** | **完全無料** | 高機能なサーバー側の処理 |
